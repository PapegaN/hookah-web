# Правила для `hookah-web`

## Роль репозитория

Фронтенд для операционного контура кальянной: дашборд, учет табака, заказы и рабочие сценарии персонала.

## Стек

- Vue 3
- Composition API
- TypeScript
- Vue Router
- Pinia
- Vite
- Vitest
- Playwright
- ESLint
- Prettier

## Локальные правила

- Компоненты держим небольшими и ориентированными на сценарии пользователя.
- Состояние выносим в Pinia, если его используют несколько экранов или оно отражает доменную модель.
- Все новые маршруты типизируем через `RouteRecordRaw`.
- Временные мок-данные держим отдельно и называем так, чтобы их было легко заменить API-данными.

## Коммиты

- Пишем на русском языке.
- В одном коммите не смешиваем визуальный рефакторинг и бизнес-логику.
- Перед коммитом запускаем `npm run check`.
